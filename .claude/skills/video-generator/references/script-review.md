# 脚本评审标准

用于 Content Agent 生成 script.md 后的质量审查。

## 评审清单（全部必须满足）

### 结构
- [ ] 总时长不超过 180 秒
- [ ] 每段 15-40 秒，开场不超过 20 秒，结尾不超过 15 秒
- [ ] 每段之间有明确的逻辑递进关系

### 内容
- [ ] 开场前3秒有"钩子"（反直觉事实/问题/数据）
- [ ] 每段至少使用1个类比或比喻
- [ ] 每段至少包含1个具体数据/年份/人名/案例
- [ ] 每段内容100字以上
- [ ] 专业术语首次出现时有解释

### 视觉
- [ ] 每段有具体的视觉描述，动画师可以直接实现
- [ ] 视觉描述包含具体的元素、位置、动作
- [ ] 没有模糊的"展示相关画面"类描述

### 语言
- [ ] 口语化，像在跟朋友聊天
- [ ] 句子短，避免从句套从句
- [ ] 用"你"、"我们"拉近距离
- [ ] 没有"综上所述"、"总而言之"等过渡词
- [ ] 直接说结论，不绕弯子

### 结尾
- [ ] 有行动号召或思考问题
- [ ] 不给观众留下"所以呢？"的感觉

## 优化原则

发现问题后：
1. **优先级**：结构问题 > 内容问题 > 语言问题
2. **改动最小化**：只改有问题的段落，不重写整个脚本
3. **保持原有风格**：不改变作者的声音和表达方式
4. **验证修改**：修改后重新检查时长是否仍匹配
