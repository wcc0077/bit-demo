# Animation Review Standards

Used to review the quality of animation code generated by the Manim Agent.

## Checklist (all must be satisfied)

### Timing Match
- [ ] Each section's `self.wait()` + `Σ(self.play run_time)` equals the script section duration (±0.5s)
- [ ] Total duration matches the voiceover total duration

### Layout (No Overlapping)
- [ ] Title uses `to_edge(UP, buff=0.5)`, fixed at top
- [ ] Content uses `next_to(title, DOWN, buff>=0.3)` for auto-alignment
- [ ] Left-right element spacing >= 1.0 units
- [ ] **Bottommost element** must use `to_edge(DOWN, buff=0.8)` (never use `shift(DOWN)`)
- [ ] Second-from-bottom elements use `next_to(bottom_element, UP, buff=0.3)` or batch into `VGroup(...).arrange(DOWN, buff=0.3)` then `to_edge(DOWN, buff=0.8)`
- [ ] No use of `shift()` for hard-positioning that causes overlap

> Why: The subtitle area occupies the bottom ~80px (y ≈ -1.75 to -1.375). Any element hard-positioned with `shift(DOWN * N)` to y < -1.0 will be covered by subtitles.

### Mobile Readability (New)
- [ ] **Body text font_size >= 20** (`Text("body", font_size=20, ...)` is the minimum)
- [ ] **Title font_size >= 36** (ensure top title is legible on phone)
- [ ] **Label/annotation font_size >= 14** (axis labels, arrow annotations, etc.)
- [ ] **Number font_size >= 16** (vector values, bit-weight numbers, etc.)
- [ ] Table/list row spacing >= 0.1 (use `arrange(DOWN, buff>=0.1)`)
- [ ] In two-column layouts, each column width <= half screen minus 0.5 units
- [ ] No text with `font_size < 14` anywhere

### Arrows & Connections
- [ ] Arrow start and end points align to element edges (`buff=0` or `buff=0.05`)
- [ ] Arrow length >= 0.8 units (too short is unreadable)
- [ ] Parallel arrows spaced >= 0.3 units apart
- [ ] Arrow labels placed via `next_to(arrow, UP/DOWN/LEFT/RIGHT, buff=0.05)`
- [ ] No arrows passing through text or other graphical elements
- [ ] Bidirectional relationships use two separate arrows, not one line

> Why: On small phone screens, short arrows are hard to read and crossings create confusion.

### Visual Elements (Rich & Expressive)
- [ ] At least 1 **non-box/circle** visual element per scene (Unicode icons, line art, `MathTex` formulas, `FunctionGraph` waveforms, `Star`, `Triangle`, etc.)
- [ ] Concept concretization: music with `♪`, gears with `⚙`, sound with `FunctionGraph` waveform, data with `BarChart`, sun with line art
- [ ] No 3 consecutive scenes using only `Circle` and `RoundedRectangle`
- [ ] Used `MathTex` or `Text` to express mathematical relationships (when numbers, units, ratios are involved)
- [ ] At least 1 emphasis effect (`Indicate`, `Circumscribe`, `Flash`, etc.)
- [ ] Line art is clean (5-15 lines), unified style (stroke-only or low-opacity fill)

### Color Palette
- [ ] Uses defined constants (BG_COLOR, TITLE_COLOR, TECH_COLOR, etc.)
- [ ] No hardcoded undefined color values (except decorative elements)
- [ ] Keyword highlight colors match semantics

### Animation Pacing
- [ ] No simultaneous display of all elements (progressive `self.play()`)
- [ ] Key content appears first, details follow
- [ ] Scene endings include `FadeOut(*self.mobjects)` transition

### Code Structure
- [ ] Each scene class is independent, no shared state
- [ ] Class names follow format: `Scene01`, `Scene02`...
- [ ] `self.camera.background_color = BG_COLOR` as first line of each scene
- [ ] No TODO or incomplete code

### Render Safety (see `pre-render-checks.md` for details)
- [ ] `np.array()` / `move_to()` use pure numeric coordinates, no Manim vector constant mixing
- [ ] No chained `.next_to()` calls inside VGroup constructors
- [ ] No use of `axes.get_graph(x_range=...)` (use `axes.plot()` or Dot instead)
- [ ] No `MathTex` containing Chinese characters (use `Text()` instead)
- [ ] Color constants within defined range (no use of undefined colors like `MAGENTA`)

## Common Issues

| Issue | Fix |
|-------|-----|
| Element overlap | Use `arrange()` / `next_to()` instead of `shift()` |
| Timing mismatch | Adjust the last `self.wait()` value |
| Bottom covered by subtitles | Use `to_edge(DOWN, buff=0.8)` for bottommost elements, **never** `shift(DOWN * N)` |
| Multi-line bottom text overlap | Use `VGroup(...).arrange(DOWN, buff=0.3).to_edge(DOWN, buff=0.8)` |
| Too much shown at once | Split into multiple `self.play()` for progressive reveal |
| Inconsistent colors | Replace hardcoded values with constants |
| Monotonous visuals | Introduce Unicode icons (♪ ⚙ ⚡ ☁ ✓), line art, `FunctionGraph`, etc. |
| Only boxes and circles | Add `Star`, `Triangle`, `Annulus`, `Arc`, `Dot`, `Brace`, line art |
| Line art too complex | Limit to 5-15 lines, remove redundant details, keep abstract style |
| Text too small on phone | Body < 20 → 20+, labels < 14 → 14+ |
| Arrows too short or crossing | Length >= 0.8 units, parallel spacing >= 0.3, labels via `next_to` |
| Two-column content squeezed | Reduce text size or fewer columns, ensure single-side width <= half screen minus 0.5 |
